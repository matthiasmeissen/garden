# Garden

Things I do and need to remember.

## What is this

A personal knowledge base built from practice, experimentation, and LLM-assisted learning. Not meant to be complete — meant to be useful.

## Structure

Two main domains, three levels deep maximum.

**computation/** — Rust, Embedded Systems, DSP, Shaders, TouchDesigner, Game Engines, Web

**making/** — Woodworking, Electronics, Music Making and Synthesis

**principles-and-philosophy.md** — the thinking behind how I approach learning and building

Each subdomain has an `_index.md` with a learning progression, key resources, and an ideas backlog.

## How It Works

Content is extracted from Claude Code sessions, research conversations, and hands-on practice. Key learnings get written into the relevant subdomain file. Over time the vault becomes a queryable memory of everything worked through.

Tags connect tools and concepts across domains — `#faust`, `#eurorack`, `#rust`, `#embedded` etc. — without forcing things into rigid folders.

## Principles

Do more with less. Start making before you feel ready. Foundational texts over blog posts. Cross-references over rigid categories.
